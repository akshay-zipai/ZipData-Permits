# Bedrock Integration Notes

This document explains how to use the offline dataset generated by
`scripts/generate_offline_dataset.py` with Amazon Bedrock and a retrieval
layer.

Files produced by the generator:

- `output/offline_dataset.jsonl` â€” newline-delimited JSON records with fields:
  `id`, `chunk_id`, `county_name`, `source_url`, `title`, `content`, `word_count`, `embedding`.
- `output/offline_dataset.parquet` â€” optional Parquet snapshot (compact, columnar).
- `output/manifest.json` â€” local manifest listing produced files.
- `output/manifest_s3.json` â€” S3 manifest referencing `s3://` URIs (if uploaded).

Recommended approaches to connect to Bedrock
------------------------------------------

1) Minimal / low-cost (S3 + lightweight retriever)

- Upload the JSONL (and Parquet) to S3 and give your EC2 (or Lambda) role access.
- Implement a small retriever service (container or Lambda) that:
  - Loads embeddings from the JSONL (or parquet) into memory or a lightweight index.
  - Computes the query embedding (locally using the same embedding model or via Bedrock if available).
  - Computes cosine similarities and returns top-K chunks.
  - Calls Bedrock's model with those retrieved chunks included in the prompt.

Advantages: simple, reproducible, minimal infra.

2) Production / scale (managed retrieval + Bedrock)

- Ingest the JSONL into a managed retrieval service (Amazon Kendra, OpenSearch k-NN, Pinecone, Milvus, etc.).
- Use metadata filters (e.g. `county_name`) for precise retrieval.
- At query time, call the retrieval API, then pass the top-K results as context to Bedrock for generation.

Advantages: faster, more reliable, supports larger datasets and metadata filtering.

S3 IAM and security
-------------------

- Attach an IAM role to EC2 with least-privilege S3 access for the bucket:

  - `s3:PutObject`, `s3:GetObject`, `s3:ListBucket`, `s3:DeleteObject` for the target bucket/prefix.
  - If using SSE-KMS, add `kms:Decrypt` and `kms:Encrypt` for the KMS key.

- Use bucket policies and encryption to restrict access.

Example retriever + Bedrock call (pseudo)
----------------------------------------

1. Load top-K chunks (after computing query embedding):

```
# Pseudocode
query_vec = embedder.embed_single(query)
scores = cosine_similarity_matrix(query_vec, dataset_embeddings)
top_idx = argsort(scores)[-k:]
context = "\n\n---\n\n".join(dataset[i]["content"] for i in top_idx)
prompt = f"Use the following context:\n{context}\nQuestion: {query}\nAnswer:"
response = bedrock_client.invoke_model(model_id='...', input_text=prompt)
```

2. Bedrock-specific note: Bedrock is a model serving platform â€” it does not provide a built-in vector store. Use a retrieval layer (Kendra/vector DB) and provide retrieved context to Bedrock when generating.
